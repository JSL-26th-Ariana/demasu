デマス

韓国のトイレが、すぐ見つかる。

韓国を訪れる旅行者のために開発した、トイレマップサービスです。
Naver Maps APIで周辺トイレをリアルタイム表示、 清潔度レビューや設備タグで最適なトイレを素早く見つけられます。

開発期間：2週間 / チーム：6名 / Spring Boot フルスタック

ABOUT

韓国のトイレ探しを
もっと快適に。

「デマス」は「どこのトイレ」に由来するサービス名です。 韓国旅行中に困りがちなトイレ探しの問題を解決するため、 地図連携・レビュー・タグフィルタ機能を一体化しました。

ユーザーによるレビュー投稿・トイレ情報提供・修正提案を通じて コミュニティ主導で情報が充実し続けるプラットフォームです。 管理者ダッシュボードで5タブ統合管理を実現しています。

FEATURES

主な機能

地図検索から管理者ダッシュボードまで、フルスタックで実装しました。

🗺️

リアルタイム地図検索

Naver Maps APIと現在地取得を組み合わせ、周辺トイレをリアルタイムで地図に表示。Haversine公式による精密な距離計算で最寄り一覧を提供。

Naver MapsGeolocationHaversine
🔍

複数タグフィルタリング

24時間・車椅子対応・おむつ交換台・清潔・多目的トイレなどのタグを複数同時選択してフィルタリング。キーワード・建物名検索にも対応。

複数選択キーワード検索
👤

会員認証

一般ログイン・Google OAuth2ソーシャルログインをサポート。パスワード忘れ時のID＋メール認証による3ステップ再設定フローも実装。

Spring SecurityGoogle OAuth2BCrypt
⭐

レビュー投稿・管理

清潔度評価・テキスト・画像アップロードに対応したレビュー機能。最新順／高評価順の並び替えと不正レビュー報告機能も搭載。

画像アップロード報告・非表示
🚽

トイレ情報提供・修正提案

ユーザーがトイレの新規登録・修正提案が可能。Naver地図クリックで座標を自動入力。管理者の承認後に地図へ反映されます。

ユーザー投稿管理者承認
🛡️

管理者ダッシュボード

会員・報告・トイレ承認・お問い合わせ・修正提案の5タブ統合管理画面。ステータスフィルタ・10件ページネーション・管理者アカウント保護を完備。

5タブ管理ROLE_ADMIN
💬

お問い合わせ

ユーザーからの問い合わせ送受信・一覧・詳細閲覧。管理者が回答を入力するとステータスがANSWEREDへ更新されます。

WAITING / ANSWERED
📋

マイページ

プロフィール編集・パスワード変更・アイコン選択・最近見たトイレ履歴・マイレビュー・マイ問い合わせ一覧をひとまとめに管理。

画像アップロード閲覧履歴
🔒

セキュリティ・権限管理

ROLE_USER / ROLE_ADMINによる権限分離。停止・退会アカウントのログインエラーメッセージ個別表示。管理者アカウントの停止・削除を禁止。

SUSPENDEDDELETEDアカウント保護

DESIGN

設計

システムのER図とユースケース図です。

📊 ER図
👥 ユースケース
🏗 アーキテクチャ
👤 一般ユーザー（ROLE_USER）
  • 周辺トイレを地図で検索・確認する
  • タグ・キーワードでトイレをフィルタリングする
  • トイレの詳細情報・レビューを閲覧する
  • レビューを投稿・削除する（画像付き）
  • 不正レビューを報告する
  • 新規トイレを提案登録する
  • 既存トイレの修正提案を送る
  • お問い合わせを送信・確認する
  • マイページでプロフィールを編集する
  • 最近見たトイレ履歴を確認する
  • パスワードをリセットする（メール認証）
  • Google OAuthでソーシャルログインする
🛡️ 管理者（ROLE_ADMIN）
  • 管理者ダッシュボードにアクセスする
  • 会員一覧・詳細を確認する
  • 会員を停止・復元・削除する（管理者除く）
  • ユーザー提報トイレを承認・却下する
  • 報告されたレビューを非表示・却下する
  • お問い合わせに回答する
  • 修正提案を承認・却下する
  • 各機能のページネーション・フィルタを操作する
🔓 未ログインユーザー（Guest）
  • トイレ地図・一覧を閲覧する
  • トイレ詳細・レビューを読む
  • 会員登録・ログインページにアクセスする
  • Google OAuthログインを開始する
🔄 システム（自動処理）
  • 公共データAPIからトイレ情報をインポートする
  • Haversine公式で距離を自動計算する
  • BCryptでパスワードを暗号化する
  • メール認証トークンを送信・検証する
  • 最近見たトイレを自動記録する

TROUBLESHOOTING

技術的な課題と解決

開発中に直面した主な技術的課題と、その解決プロセスです。

ISSUE 01

AdminPageResponse DTO の設計 — フィールド名の不一致問題

🔴 PROBLEM
共通の PageResponse<T> は items / page フィールド名を使用していたが、 dashboard.html の JavaScript が content / number を期待しており、 フロントエンドでデータが一切表示されない問題が発生。
🟢 SOLUTION
Admin 専用の AdminPageResponse<T> DTO を新規作成し、 content / number / totalPages / size / totalCount フィールド名でフロントエンドの期待値に統一。 既存の共通コードを変更せずに解決。
ISSUE 02

Thymeleaf Boolean Wrapper の NullPointerException

🔴 PROBLEM
th:if="${!toilet.isWheelchair}" の式評価時、Boolean Wrapper 型が null の場合に EL1001E: Type conversion problem が発生。Spring EL では null の Boolean を ! で反転しようとするとエラーになる。
🟢 SOLUTION
!= true パターンへ変更。 th:if="${toilet.isWheelchair != true}" とすることで、 null と false どちらの場合も安全に処理できる null-safe な書き方を採用。
ISSUE 03

公共 API データが管理者画面に大量表示される問題

🔴 PROBLEM
政府の公共データ API で取り込んだ数万件のトイレデータが 管理者の「トイレ承認」タブに全件表示され、ページネーションが 800 ページ超に。 ユーザー提報データのみを管理対象とすべきところが混在していた。
🟢 SOLUTION
toilets テーブルの source カラムを活用し、 AdminMapper.xml の WHERE 句に AND source = 'USER' 条件を追加。 ユーザー提報データのみを管理画面へ表示するよう修正し解決。
ISSUE 04

ログインエラーメッセージの状態別分岐

🔴 PROBLEM
Spring Security のデフォルト実装では、停止アカウント・退会アカウント・ パスワード不正を区別できず、すべて同一のエラーが表示されていた。 UX として状態に応じたメッセージが必要だった。
🟢 SOLUTION
CustomUserDetails で isEnabled()（SUSPENDED → false）と isAccountNonLocked()（DELETED → false）を分離。 カスタム AuthenticationFailureHandler で例外種別ごとに ?error=suspended / ?error=deleted パラメータを付与し、 フロントエンドで状態別トーストメッセージを表示。
